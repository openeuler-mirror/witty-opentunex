# mem-bottleneck

OS级别内存瓶颈分析Skill，分析内存利用率、Swap活动、页错误等。

## 总体功能

分析OS级别内存性能瓶颈：

1. **环境检查** - NUMA、内存总量、内核参数
2. **指标收集** - vmstat/sar/slabtop
3. **瓶颈识别** - 内存压力、Swap颠簸、Slab问题
4. **NUMA分析** - 跨节点内存访问

**Scope**: 仅分析OS级别内存资源

## 前置依赖

### 必需依赖

- **remote-execution skill** - SSH远程连接

### 工具依赖

- `vmstat`, `sar`, `free`

## 用法

### 自然语言输入示例

```
分析192.168.1.100的内存瓶颈
```

```
系统内存使用率高，帮我看看原因
```

### 输出示例

```markdown
## 内存瓶颈分析报告

### 环境信息
- 内存: 64GB
- Swap: 8GB
- NUMA: 2节点

### 指标分析

| 指标 | 当前值 | 阈值 | 状态 |
|------|--------|------|------|
| used% | 92% | 90% | ⚠️ 压力 |
| swap% | 25% | 10% | ❌ 异常 |
| majflt/s | 1200 | 1000 | ❌ 颠簸 |

### 瓶颈识别

| 瓶颈类型 | 证据 |
|----------|------|
| 内存压力 | used%=92% |
| Swap活动 | si/so > 10MB/s |
| 页错误 | majflt=1200/s |

### 优化建议
1. 降低vm.swappiness
2. 增加内存或清理缓存
```
